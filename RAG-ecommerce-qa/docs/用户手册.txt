================================================================================
  1. 项目简介
================================================================================

  本系统是一个智能电商客服问答平台。当你输入问题时，系统会：

    ① 把你的问题用 BGE 模型转成"向量"（一串数字，代表语义）
    ② 在知识库中搜索最相似的问答对
    ③ 把找到的相关内容作为"参考资料"喂给 DeepSeek 大模型
    ④ 大模型基于参考资料生成专业回答，避免瞎编

  核心功能：
    - 智能问答（9 大分类：物流配送、退换货、订单支付、优惠活动、
      产品咨询、账户服务、售前咨询、售后服务、其他）
    - 文档上传（支持 PDF / DOCX / TXT）
    - 推荐问题（自动推荐相关问题）
    - 电商风格 Web 界面

================================================================================
  2. 环境要求
================================================================================

  必需软件：
    - Python 3.10（推荐使用 Anaconda 管理）
    - Windows 10/11 或 Linux（本系统在 Windows 上开发测试）
    - 网络连接（用于调用 DeepSeek API）

  硬件建议：
    - 内存：4GB 以上（首次运行需下载约 400MB 的向量模型）
    - 磁盘：2GB 以上空闲空间（向量数据库 + 数据文件）

================================================================================
  3. 安装与配置
================================================================================

  第一步：安装 Python 依赖

    打开终端（cmd 或 PowerShell），进入项目目录，执行：

      cd d:\AIProject\RAG
      pip install -r requirements.txt

    说明：
    - 首次安装会下载 sentence-transformers（约 400MB），请耐心等待
    - torch 会自动安装 CPU 版本，如需 GPU 加速请另行配置
    - 如果下载慢，可以用国内镜像：
      pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

  第二步：设置环境变量（关键步骤！）

    每次启动前，必须先设置这个环境变量，否则会尝试联网下载模型失败：

      Windows CMD:
        set HF_HUB_OFFLINE=1

      Windows PowerShell:
        $env:HF_HUB_OFFLINE=1

      Linux/Mac:
        export HF_HUB_OFFLINE=1

    这个环境变量的作用是：告诉 sentence-transformers 使用本地缓存的模型，
    而不是去 HuggingFace 下载。模型首次安装时会自动缓存到本地。

  第三步：配置 API 密钥（可选）

    系统默认使用内置的 DeepSeek API 密钥。如果需要更换自己的密钥，
    编辑 api.py 文件的第 31 行：

      DEEPSEEK_API_KEY = "你的密钥"

================================================================================
  4. 数据准备
================================================================================

  系统需要两类数据：

  ★ 结构化 Q&A 数据（核心知识库）
    位置：data/ecommerce_qa.json（约 8000 条电商客服问答对，已内置）

    如果你想从原始对话数据重新生成，执行：
      python data_loader.py

    注意：data_loader.py 需要原始训练数据文件（train.txt），
    原始数据路径在脚本中配置（第 143 行 DATA_DIR 变量）。

  ★ 自定义文档（可选）
    你可以通过 Web 界面上传 PDF/DOCX/TXT 文档，系统会自动解析入库。

================================================================================
  5. 知识库初始化
================================================================================

  知识库是将 Q&A 数据转为向量存入 ChromaDB 的过程。

  初始化方法一：通过脚本初始化
    执行：
      python init_knowledge_base.py

    参数说明（可选）：
      --data JSON文件路径       （默认：data/ecommerce_qa.json）
      --batch-size 批处理大小   （默认：64）
      --max-items 最大导入数    （调试用，如 --max-items 100）

    示例：
      python init_knowledge_base.py --max-items 1000 --batch-size 32

  初始化方法二：通过 API 初始化
    启动系统后，调用 API 初始化（见第 8 节）。

  说明：
    - 向量数据存储在 vector_store_v2/ 目录
    - 如果已有向量数据，重新初始化会覆盖旧数据
    - 初始化 8000 条数据大约需要 2-5 分钟（取决于 CPU 性能）

================================================================================
  6. 启动系统
================================================================================

  完整启动流程（首次使用）：

    # 步骤 1：进入项目目录
    cd d:\AIProject\RAG

    # 步骤 2：设置离线模式环境变量
    set HF_HUB_OFFLINE=1

    # 步骤 3：（首次）初始化知识库
    python init_knowledge_base.py

    # 步骤 4：启动 API 服务器
    python api.py

  日常启动（知识库已有数据）：

    cd d:\AIProject\RAG
    set HF_HUB_OFFLINE=1
    python api.py

  启动成功后，你会看到类似输出：
    INFO:     Uvicorn running on http://0.0.0.0:8084

  然后在浏览器中打开：http://localhost:8084

================================================================================
  7. 使用指南
================================================================================

  网页界面使用：

    ① 打开浏览器，访问 http://localhost:8084
    ② 在输入框中输入问题（如"什么时候发货？"）
    ③ 可以选择分类来缩小检索范围
    ④ 点击发送按钮或按回车键
    ⑤ 系统会返回 AI 生成的回答，并显示参考来源

  文档上传：

    ① 在网页界面找到上传按钮
    ② 选择 PDF / DOCX / TXT 文件
    ③ 点击上传，系统自动解析并加入知识库
    ④ 上传的文档内容可以被后续问答检索到

  推荐问题：

    ① 系统首页会展示推荐问题，点击即可快速提问
    ② 每次问答后会推荐同类别的相关问题

================================================================================
  8. API 接口说明
================================================================================

  如果你需要通过代码调用系统，可以使用以下 API：

  ┌──────────┬────────┬──────────────────────────────────────┐
  │ 接口     │ 方法   │ 说明                                  │
  ├──────────┼────────┼──────────────────────────────────────┤
  │ /        │ GET    │ 返回前端页面                          │
  │ /query   │ POST   │ 智能问答（核心接口）                   │
  │ /upload  │ POST   │ 上传文档（PDF/DOCX/TXT）              │
  │ /init    │ POST   │ 初始化知识库                          │
  │ /clear   │ DELETE │ 清空知识库                            │
  │ /categories│ GET  │ 获取分类列表及数量                     │
  │ /suggest │ GET    │ 获取推荐问题                          │
  │ /stats   │ GET    │ 获取系统统计信息                       │
  └──────────┴────────┴──────────────────────────────────────┘

  示例：调用问答接口

    curl -X POST http://localhost:8084/query \
      -H "Content-Type: application/json" \
      -d '{"question": "什么时候发货？", "top_k": 3}'

  返回格式：
    {
      "answer": "亲，一般情况下...",
      "sources": [
        {
          "text": "问题：...\n回答：...",
          "score": 0.85,
          "category": "物流配送",
          "source": "qa_knowledge"
        }
      ],
      "category": "物流配送"
    }

  示例：上传文档

    curl -X POST http://localhost:8084/upload \
      -F "file=@/path/to/document.pdf"

  9. 常见问题
================================================================================

  Q: 启动时报错 "sentence_transformers model not found"？
  A: 检查是否设置了环境变量 HF_HUB_OFFLINE=1。
     如果已设置但仍报错，说明模型未下载，先临时去掉该环境变量
     运行一次让系统自动下载模型，之后再加上。

  Q: 页面打开了但问答没反应？
  A: 检查 DeepSeek API 是否可用。查看终端输出的错误日志。
     如果 API 不可用，系统会自动使用知识库模板回答（精确匹配模式）。

  Q: 知识库初始化很慢？
  A: 主要耗时在向量化（BGE 模型推理）。8000 条数据约需 2-5 分钟。
     可以用 --max-items 100 先测试少量数据。

  Q: 端口 8084 被占用？
  A: 编辑 api.py 最后一行，修改 port 参数为其他端口（如 8085）。

  Q: 如何添加自己的知识库数据？
  A: 方法一：通过 Web 界面上传文档（PDF/DOCX/TXT）
     方法二：编辑 data/ecommerce_qa.json 添加自定义 Q&A，然后重新初始化

  Q: ChromaDB 报错？
  A: 尝试清空 vector_store_v2/ 目录，重新运行 init_knowledge_base.py。

================================================================================

  RAG/
  ├── api.py                      # ★ 主程序 - FastAPI 服务器
  ├── requirements.txt            # Python 依赖清单

  ├── data/                       # 数据目录

  │
  ├── src/                        # 核心模块
  │   ├── embedder.py             # BGE 向量化模型
  │   ├── vector_store.py         # ChromaDB 向量存储
  │   ├── retriever.py            # 检索与生成
  │   └── document_parser.py      # PDF/DOCX/TXT 文档解析
  │
  ├── static/
  │   └── index.html              # Web 前端页面
  │
  ├── uploads/                    # 上传文档存储目录
  ├── vector_store_v2/            # ★ 向量数据库存储（核心数据）

  └── 工具脚本
      ├── data_loader.py          # 原始数据处理脚本
      ├── init_knowledge_base.py  # 知识库初始化脚本
      ├── process_brand_data.py   # 品牌数据处理
      ├── process_custom_data.py  # 自定义数据处理
      └── import_data.py          # 数据导入工具
